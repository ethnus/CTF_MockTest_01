Ethnus AWS Mock Test Project

Introduction

This challenge places you in a simulated AWS environment designed to test your ability to implement, secure, and troubleshoot a serverless architecture under realistic conditions.

  • Operate within the constraints of AWS Academy Learner Lab (limited permissions, restricted IAM role).
  • Diagnose and resolve misconfigurations across compute, storage, network, and access control.
  • Apply AWS best practices and critical thinking to identify gaps.
  • Develop skills in troubleshooting and architecting resilient cloud-native systems.
Important:
  • Do not inspect the bootstrap (deploy.sh) or evaluation (eval.sh) scripts.
  • Focus on discovery, reasoning, and resolution rather than shortcuts.
  • The objective is to build the mindset of a cloud architect and problem solver.

Resources and Purpose

Resource Purpose
S3 Bucket* Holds application data; requires encryption and restricted access.
DynamoDB Table* Stores key-value records; accessible only via private endpoints.
Lambda (Writer)* Processes incoming data, updates DynamoDB, stores objects in S3, triggers SNS.
Lambda (Reader)* Fetches and returns data via API Gateway; final output includes the success flag.
API Gateway* Private REST API exposing read operations, restricted to VPCE.
SNS Topic* Handles internal notifications for asynchronous tasks.
VPC A & B Separate networks for application components and peer services.
KMS Key* Encrypts stored data and Lambda environment variables.
EventBridge Rule* Ensures periodic activity and health verification.
